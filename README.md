# 1C AI

AI-ассистент для 1С с использованием локальных LLM, Qdrant и n8n.

Проект предназначен для построения интеллектуального помощника, который может получать данные из 1С, выполнять семантический поиск по документам и отвечать на вопросы пользователя с помощью локальной языковой модели.

## Архитектура

```text
Пользователь
      │
      ▼
     1С
(ЧатБот.epf)
      │
      ▼
     n8n
(оркестрация)
      │
 ┌────┴────┐
 ▼         ▼
Qdrant   Ollama
Вектора   LLM
      │
      ▼
 Ответ пользователю
```

## Компоненты

### 1С

Файл:

```text
ЧатБот.epf
```

Внешняя обработка для взаимодействия с AI-инфраструктурой.

### n8n

Используется для:

* приема запросов от 1С;
* обработки данных;
* построения RAG-пайплайнов;
* интеграции с внешними сервисами;
* управления AI-агентами.

### Qdrant

Векторная база данных для:

* хранения эмбеддингов;
* семантического поиска;
* реализации Retrieval-Augmented Generation (RAG).

## Быстрый старт

### Требования

* Docker
* Docker Compose
* 1С:Предприятие 8.3
* Ollama (локально)

### Запуск

```bash
docker compose up -d
```

После запуска будут доступны:

| Сервис | URL                   |
| ------ | --------------------- |
| n8n    | http://localhost:5678 |
| Qdrant | http://localhost:6333 |

## Docker Compose

Проект использует следующие сервисы:

### n8n

* Порт: 5678
* Базовая авторизация включена

### Qdrant

* Порт: 6333
* Данные сохраняются в каталоге `qdrant_data`

## Подключение Ollama

Установить Ollama:

```bash
curl -fsSL https://ollama.com/install.sh | sh
```

Загрузить модель:

```bash
ollama pull qwen3:8b
```

или

```bash
ollama pull llama3.1:8b
```

Проверить работу:

```bash
ollama run qwen3:8b
```

## Интеграция с 1С

1. Открыть обработку `ЧатБот.epf`.
2. Настроить адрес n8n Workflow.
3. Выполнить тестовый запрос.
4. Проверить получение ответа от AI-сервиса.
